1. Current heuristic uses inverse difference which may over-penalize near-capacity solutions.
2. Introduce exponential decay for smoother transitions near capacity limits.
3. Add size ratio consideration to better handle varying item sizes.
4. Maintain numerical stability while reducing sensitivity to exact constraint matching.}  

```python
import numpy as np 
def heuristics_v2(node_attr, node_constraint):
    #EVOLVE-START
    sizes = node_attr if node_attr.ndim == 1 else node_attr[:,0]
    n = len(sizes)
    size_ratios = np.minimum(sizes.reshape(n,1)/sizes, sizes/sizes.reshape(n,1))
    size_matrix = sizes.reshape(n,1) + sizes
    heuristic = size_ratios * np.exp(-0.5*np.abs(size_matrix - node_constraint))
    np.fill_diagonal(heuristic, 0)
    return heuristic/heuristic.max()
    #EVOLVE-END
```
