1. Current version combines inverse squared distance and log scaling but may over-dampen proximity signals.
2. Need smoother distance-to-weight mapping while preserving numerical stability.
3. Solution: Use softplus for log scaling and balance terms with square root.}     
```python
import numpy as np 
def heuristics_v2(distance_matrix):
    #EVOLVE-START
    epsilon = 1e-15
    softplus_weights = np.log1p(np.exp(distance_matrix)) 
    return np.sqrt(1/(distance_matrix + epsilon)) * (1/softplus_weights)
    #EVOLVE-END       
    return 1 / distance_matrix
```
