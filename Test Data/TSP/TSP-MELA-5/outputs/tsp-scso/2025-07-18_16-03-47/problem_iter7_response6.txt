{thought process:
1. Current version uses log and inverse squared terms but may over-dampen mid-range distances.
2. Need smoother distance-weight mapping while maintaining stability.
3. Solution: Use softplus for smoother log behavior and balance inverse terms with sqrt for gentler decay.}     
```python
import numpy as np 
def heuristics_v2(distance_matrix):
    #EVOLVE-START
    epsilon = 1e-15
    softplus_weights = np.log1p(np.exp(distance_matrix))
    return (1 / np.sqrt(distance_matrix + epsilon)) * (1 / softplus_weights)
    #EVOLVE-END       
    return 1 / distance_matrix
```
