1. The error occurs because the distance matrix contains negative values or zeros, causing the logarithm to return NaN (undefined)
2. We need to ensure the distance matrix is strictly positive before applying the logarithm
3. We should also handle very small distances to avoid numerical instability
4. The final result needs to be properly normalized to form a valid probability distribution (Simplex constraint)}

```python
import numpy as np
def heuristics_v2(distance_matrix):
    #EVOLVE-START
    # Ensure distances are positive and avoid numerical issues
    epsilon = 1e-10
    safe_distances = np.maximum(distance_matrix, epsilon)
    
    # Calculate heuristic values
    heuristic_values = np.sqrt(np.log(safe_distances))
    
    # Convert to probabilities by softmax normalization
    exp_values = np.exp(heuristic_values - np.max(heuristic_values, axis=1, keepdims=True))
    probabilities = exp_values / np.sum(exp_values, axis=1, keepdims=True)
    
    return probabilities
    #EVOLVE-END
```
