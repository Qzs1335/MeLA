1. Current version uses inverse squared distance with epsilon for stability.
2. To improve non-linearity while maintaining simplicity, we can blend linear and inverse terms.
3. Adding a small linear term (0.01*distance) will preserve proximity information while preventing dominance of extreme values.}     
```python
import numpy as np 
def heuristics_v2(distance_matrix):
    #EVOLVE-START
    epsilon = 1e-15
    blended = 0.99/(distance_matrix**2 + epsilon) + 0.01/distance_matrix
    return blended
    #EVOLVE-END       
    return 1 / distance_matrix
```
