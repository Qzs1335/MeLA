1. Current version combines inverse square and log weights well but may over-penalize large distances
2. Need smoother distance scaling while maintaining stability
3. Solution: Use softened inverse (1/(x+1)) for better gradient and add epsilon inside log
4. Keep multiplicative combination for balanced weighting}     
```python
import numpy as np 
def heuristics_v2(distance_matrix):
    #EVOLVE-START
    epsilon = 1e-15
    soft_inv = 1/(distance_matrix + 1)
    log_weights = np.log(distance_matrix + 1 + epsilon)
    return soft_inv * (1/log_weights)
    #EVOLVE-END       
    return 1 / distance_matrix
```
