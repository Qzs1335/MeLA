{thought process:
1. Current heuristic uses simple inverse distance which may miss local optima.
2. Adding logarithmic scaling could better handle varying distance scales.
3. Introducing a small epsilon prevents division by zero.
4. Combining inverse with square root captures non-linear relationships.}

```python
import numpy as np
def heuristics_v1(distance_matrix):
    #EVOLVE-START
    epsilon = 1e-10
    return np.sqrt(1 / (distance_matrix + epsilon)) + np.log(1 + 1/distance_matrix)
    #EVOLVE-END       
    return 1 / distance_matrix
```
