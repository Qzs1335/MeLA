1. Current version uses log + inverse + power-law but may over-penalize large distances.
2. Simpler log transformation with linear distance maintains sensitivity while being more stable.
3. Added clipping to prevent extreme values and ensure numerical stability.}

```python
import numpy as np
def heuristics_v2(distance_matrix):
    #EVOLVE-START
    epsilon = 1e-10
    clipped_dist = np.clip(distance_matrix, epsilon, None)
    return np.log(1 + 1/clipped_dist)
    #EVOLVE-END
    return 1 / distance_matrix
```
